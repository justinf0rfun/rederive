# Local redo Bundle Import

The MVP authoring path is:

1. Generate the case locally with the mature `redo` skill.
2. Upload the generated Markdown file in `/admin`, or convert it into
   `redo_bundle_v1` JSON for stricter control.
3. Paste/upload the artifact into `/admin` under `local redo import`.
4. Review imported modules.
5. Publish through the existing immutable publish flow.

The website does not call the local skill runtime. It receives structured data,
validates it, and stores it as a reviewable generation run.

## Markdown Import

The admin page accepts `.md` files generated by the redo skill. The converter
parses:

- orientation fields at the top of the document
- `## 阶段 N：...` sections and candidate-option tables
- throughline table
- debt map tables
- pain ranking table
- causal chain code block
- reference list

If the Markdown contains non-contract debt IDs such as `D3b` or `D2-4`, the
converter maps them to clean `D<N>` IDs and preserves the original ID in the
debt text. Missing transferable-pattern and boundary modules are derived
conservatively from throughline and pain-ranking content.

## Required Shape

```json
{
  "bundleVersion": "redo_bundle_v1",
  "sourceMode": "local_redo_skill",
  "promptVersion": "redo-skill-local-2026-05-26",
  "exportedAt": "2026-05-26T00:00:00.000Z",
  "language": "zh",
  "topic": {
    "slug": "kafka",
    "displayName": "Kafka",
    "aliases": ["Apache Kafka"],
    "category": "distributed-systems"
  },
  "designQuestions": [],
  "sources": [],
  "sourceEvidence": [],
  "evidenceClaims": [],
  "orientation": {},
  "stages": [],
  "throughline": {},
  "transferablePattern": {},
  "boundaries": [],
  "debtMap": {
    "resolved": [],
    "mitigated": [],
    "unresolved": []
  },
  "painRanking": [],
  "causalChain": {}
}
```

The module fields use the same shape as the published `redo` contract in
`app/domain/redo/contract.ts`.

## Import Rules

- Sources must include at least one paper, design doc, proposal, or standard.
- Factual claims must link to source evidence.
- Inference and controversial judgment claims must link to basis claims.
- Stage options must include at least two rejected options and exactly one
  chosen option.
- Debt IDs must trace from stages into the debt map, pain ranking, and causal
  chain.
- Imported modules start as `draft`; publishing still requires module approval.

## Stored Result

Import creates:

- one `generation_runs` row with `status=ready_for_review`
- `source_documents` rows from `sources`
- `source_evidence`, `evidence_claims`, and claim links
- draft modules for orientation, stage outline, stages, throughline, pattern,
  boundaries, debt map, pain ranking, causal chain, and source notes
- an audit entry recording the admin import action
